## CSV Annotation Files

### File Naming Convention

CSV files follow this naming pattern:
```
how2sign_realigned_{split}_preprocessed_fps{suffix}.csv
```

Where:
- `{split}`: `train`, `val`, or `test`
- `{suffix}`: Optional suffix indicating the variant:
  - `` (empty): Original split with raw text annotations
  - `_filtered`: Balanced/filtered split with raw text
  - `_gloss_qwen8b`: Gloss annotations generated by Qwen-8B model
  - `_gloss_t5`: Gloss annotations generated by T5 model

### CSV File Variants

#### 1. Original Split (`how2sign_realigned_{split}_preprocessed_fps.csv`)
- **Description**: Full dataset with raw English text annotations
- **Usage**: Default dataset for training/evaluation
- **Example**: `how2sign_realigned_train_preprocessed_fps.csv`

#### 2. Filtered Split (`how2sign_realigned_{split}_preprocessed_fps_filtered.csv`)
- **Description**: Balanced subset of the dataset
- **Usage**: Used when `cfg.DATASET.H2S.BALANCED = True`
- **Purpose**: Provides a more balanced distribution of samples
- **Example**: `how2sign_realigned_train_preprocessed_fps_filtered.csv`

#### 3. Gloss-Qwen Split (`how2sign_realigned_{split}_preprocessed_fps_gloss_qwen8b.csv`)
- **Description**: Dataset with gloss annotations generated by Qwen-8B model
- **Usage**: Used when `cfg.DATASET.H2S.GLOSS = "qwen"`
- **Purpose**: Uses gloss-level annotations instead of raw text
- **Example**: `how2sign_realigned_train_preprocessed_fps_gloss_qwen8b.csv`

#### 4. Gloss-T5 Split (`how2sign_realigned_{split}_preprocessed_fps_gloss_t5.csv`)
- **Description**: Dataset with gloss annotations generated by T5 model
- **Usage**: Used when `cfg.DATASET.H2S.GLOSS = "t5"`
- **Purpose**: Uses gloss-level annotations instead of raw text
- **Example**: `how2sign_realigned_train_preprocessed_fps_gloss_t5.csv`

### CSV Column Schema

Each CSV file contains the following columns:

| Column Name | Type | Description | Example |
|------------|------|-------------|---------|
| `SENTENCE_NAME` | string | Unique identifier for the sample/clip | `0DU7wWLK-QU_0-8-rgb_front` |
| `SENTENCE` | string | Text annotation (raw text or gloss depending on variant) | `"I love learning sign language"` or `"LOVE LEARN SIGN-LANGUAGE"` |
| `START_REALIGNED` | float | Start frame index after realignment | `12.5` |
| `END_REALIGNED` | float | End frame index after realignment | `89.3` |
| `fps` | int/float | Frame rate of the video sequence | `25` or `29.97` |
| `DURATION` | float | Duration in seconds (computed as `END_REALIGNED - START_REALIGNED`) | `76.8` |

**Note**: The `DURATION` column is computed during dataset loading:
```python
self.csv['DURATION'] = self.csv['END_REALIGNED'] - self.csv['START_REALIGNED']
```

### CSV Processing

During dataset initialization, the following filtering is applied:

```python
# Remove sequences longer than 30 seconds
self.csv = self.csv[self.csv['DURATION'] < 30].reset_index(drop=True)
```

### Bad Sample IDs

The following sample IDs are known to be broken (failed pose fitting) and are automatically excluded during loading:

## Pose Data Files

### File Format

- **Format**: Pickle (`.pkl`) files
- **Location**: `{ROOT}/{split}/poses/{SENTENCE_NAME}/{SENTENCE_NAME}_{frame_id}_3D.pkl`
- **Naming**: Sequential frame numbering starting from 0
- **Example**: `0DU7wWLK-QU_0-8-rgb_front_0_3D.pkl`, `0DU7wWLK-QU_0-8-rgb_front_1_3D.pkl`, ...

### Pickle File Content

Each pickle file contains a Python dictionary with **7 keys** representing SMPL-X parameters:

```python
{
    'smplx_root_pose': np.ndarray(shape=(3,), dtype=float32),
    'smplx_body_pose': np.ndarray(shape=(63,), dtype=float32),
    'smplx_lhand_pose': np.ndarray(shape=(45,), dtype=float32),
    'smplx_rhand_pose': np.ndarray(shape=(45,), dtype=float32),
    'smplx_jaw_pose': np.ndarray(shape=(3,), dtype=float32),
    'smplx_shape': np.ndarray(shape=(10,), dtype=float32),
    'smplx_expr': np.ndarray(shape=(10,), dtype=float32)
}
```

### SMPL-X Parameter Breakdown

| Parameter | Dimensions | Description | Joints/Parameters |
|-----------|-----------|-------------|-------------------|
| `smplx_root_pose` | 3 | Global root orientation (axis-angle) | 1 joint |
| `smplx_body_pose` | 63 | Body joint rotations (axis-angle) | 21 joints × 3 |
| `smplx_lhand_pose` | 45 | Left hand joint rotations | 15 joints × 3 |
| `smplx_rhand_pose` | 45 | Right hand joint rotations | 15 joints × 3 |
| `smplx_jaw_pose` | 3 | Jaw rotation | 1 joint |
| `smplx_shape` | 10 | Body shape parameters (β) | Shape PCA |
| `smplx_expr` | 10 | Facial expression parameters | Expression PCA |
| **Total** | **179** | Full SMPL-X representation per frame | |

### Raw Data Organization (179 dimensions)

When all SMPL-X parameters are concatenated in order:

```
Index Range | Component              | Dimensions | Description
------------|------------------------|------------|---------------------------
[0:3]       | Root pose              | 3          | Global orientation
[3:66]      | Body pose              | 63         | 21 body joints × 3
[66:111]    | Left hand pose         | 45         | 15 hand joints × 3
[111:156]   | Right hand pose        | 45         | 15 hand joints × 3
[156:159]   | Jaw pose               | 3          | Jaw rotation
[159:169]   | Shape parameters       | 10         | Body shape (β)
[169:179]   | Expression parameters  | 10         | Facial expression
------------|------------------------|------------|---------------------------
Total:                                 179 dimensions (raw)
```

## Statistics Files

### Mean and Standard Deviation Files

- **Format**: PyTorch tensor files (`.pt`)
- **Files**:
  - `h2s_mean.pt`: Mean values for z-score normalization
  - `h2s_std.pt`: Standard deviation values for z-score normalization
- **Original Shape**: `(179,)` - matches raw SMPL-X dimensions
- **Filtered Shape**: `(133,)` - after removing lower body and shape parameters

### Filtering Applied to Statistics

The same filtering applied to pose data is also applied to mean/std tensors:

```python
# Remove lower body (first 36 dimensions)
mean = mean[(3+3*11):]  # Start from index 36
std = std[(3+3*11):]

# Remove shape parameters, keep expression
mean = torch.cat([mean[:-20], mean[-10:]], dim=0)  # 179 - 36 - 10 = 133
std = torch.cat([std[:-20], std[-10:]], dim=0)
```

## Data Loading Pipeline

### Step-by-Step Loading Process

1. **CSV Loading**
   ```python
   # Load appropriate CSV based on configuration
   csv_path = "{ROOT}/{split}/re_aligned/how2sign_realigned_{split}_preprocessed_fps{suffix}.csv"
   csv = pd.read_csv(csv_path)
   ```

2. **Sample Selection**
   ```python
   # Get sample metadata
   name = csv['SENTENCE_NAME'][idx]
   fps = csv['fps'][idx]
   text = csv['SENTENCE'][idx]
   ```

3. **Frame Loading**
   ```python
   # Load all frames for this sample
   base_dir = f"{ROOT}/{split}/poses/{name}/"
   frame_list = [f"{name}_{frame_id}_3D.pkl" for frame_id in range(num_frames)]

   # Downsample if FPS > 24
   if fps > 24:
       frame_list = sample(frame_list, count=int(24*len(frame_list)/fps))
   ```

4. **Pose Assembly**
   ```python
   # Load and concatenate SMPL-X parameters
   clip_poses = np.zeros([len(frame_list), 179])
   for frame_id, frame_path in enumerate(frame_list):
       with open(frame_path, 'rb') as f:
           poses = pickle.load(f)
       pose = np.concatenate([
           poses['smplx_root_pose'],
           poses['smplx_body_pose'],
           poses['smplx_lhand_pose'],
           poses['smplx_rhand_pose'],
           poses['smplx_jaw_pose'],
           poses['smplx_shape'],
           poses['smplx_expr']
       ], axis=0)
       clip_poses[frame_id] = pose
   ```

5. **Dimension Filtering**
   ```python
   # Remove lower body joints (first 36 dims)
   clip_poses = clip_poses[:, (3+3*11):]

   # Remove shape parameters, keep expression
   clip_poses = np.concatenate([clip_poses[:, :-20], clip_poses[:, -10:]], axis=1)
   # Result: [num_frames, 133]
   ```

6. **Normalization**
   ```python
   # Z-score normalization
   clip_poses = (clip_poses - mean.numpy()) / (std.numpy() + 1e-10)
   ```

7. **Length Resampling**
   ```python
   if m_length < min_motion_length:
       # Upsample using linear interpolation
       idx_resample = np.linspace(0, m_length-1, num=min_motion_length, dtype=int)
       clip_poses = clip_poses[idx_resample]
   elif m_length > max_motion_length:
       # Downsample using linear interpolation
       idx_resample = np.linspace(0, m_length-1, num=max_motion_length, dtype=int)
       clip_poses = clip_poses[idx_resample]
   else:
       # Center crop to nearest unit_length multiple
       m_length = (m_length // unit_length) * unit_length
       idx_start = (clip_poses.shape[0] - m_length) // 2
       clip_poses = clip_poses[idx_start:idx_start + m_length]
   ```

8. **Return Format**
   ```python
   return (
       text,                              # Text annotation
       torch.from_numpy(clip_poses).float(),  # [T, 133] tensor
       m_length,                          # Sequence length
       name,                              # Sample ID
       None, None, None, None, None,     # Placeholder fields
       src                                # Dataset source ('how2sign')
   )
   ```

## Preprocessing Steps

### Summary of Transformations

| Step | Input | Output | Description |
|------|-------|--------|-------------|
| 1. FPS Resampling | Variable FPS | 24 FPS | Downsample if FPS > 24 |
| 2. Dimension Filtering | [T, 179] | [T, 133] | Remove lower body + shape params |
| 3. Z-score Normalization | [T, 133] raw | [T, 133] normalized | Apply (x - μ) / σ |
| 4. Length Resampling | [T, 133] | [T', 133] | Resample to fit length constraints |
| 5. Tensor Conversion | NumPy array | PyTorch tensor | Convert to float32 tensor |

### Final Processed Data Format (133 dimensions)

After all preprocessing, the 133 dimensions are organized as:

```
Index Range | Component              | Dimensions | Description
------------|------------------------|------------|---------------------------
[0:30]      | Upper body joints      | 30         | 10 body joints (joints 12-21)
[30:75]     | Left hand pose         | 45         | 15 hand joints × 3
[75:120]    | Right hand pose        | 45         | 15 hand joints × 3
[120:123]   | Jaw pose               | 3          | Jaw rotation
[123:133]   | Facial expression      | 10         | Expression parameters
------------|------------------------|------------|---------------------------
Total:                                 133 dimensions (processed)
```

**Body Part Distribution:**
- Upper body: 30 dims (22.6%)
- Hands: 90 dims (67.7%)
- Face: 13 dims (9.8%)

This representation focuses on **upper body, hands, and face** - the most important body parts for sign language communication, while discarding the lower body and body shape parameters.

## Notes

1. **Minimum Sequence Length**: Sequences with fewer than 4 frames are automatically filtered out
2. **Maximum Duration**: Sequences longer than 30 seconds are filtered out during CSV loading
3. **FPS Standardization**: All sequences are resampled to 24 FPS if original FPS is higher
4. **Bad Samples**: A hardcoded list of broken sample IDs is excluded during loading
5. **Normalization**: All pose data is z-score normalized using pre-computed dataset statistics
6. **Data Type**: All returned pose data is in PyTorch float32 tensor format
