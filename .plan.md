# Plan: Adapt LaMP + T2M Architecture to MotionGPT Workspace

## Overview

Adapt the full LaMP (Language-Motion Pretraining) + T2M (Masked Transformer) pipeline from `/home/student/hwu/Workplace/LaMP` into this MotionGPT codebase, integrating it alongside the existing hierarchical RVQ-GPT architecture.

This creates a **new two-stage training pipeline**:
1. **Stage: LaMP** — QFormer-based contrastive pretraining that aligns motion and text representations (4 loss objectives: contrastive, matching, language modeling, motion generation)
2. **Stage: Masked T2M** — Masked Transformer that generates motion tokens via iterative refinement using the frozen QFormer from LaMP as its text encoder

## Key Adaptation Decisions

- **Motion encoder**: Reuse the existing `RVQVae` encoder (frozen) instead of LaMP's original single-VQ encoder. The encoder outputs `[B, code_dim, T']` which we project to `1408-dim` for QFormer cross-attention (same pattern as LaMP).
- **Motion tokenization**: For T2M masked generation, use Q0 codes only from the RVQ-VAE (single quantizer, 512 tokens), decoded via `decode_q0_only()`. This matches LaMP's flat 512-token design.
- **Input features**: 133-dim (How2Sign full-body) instead of LaMP's 263-dim (HumanML3D). The VQ-VAE encoder handles this transparently.
- **QFormer**: Port the full BERT-based QFormer from LaMP (with cross-attention). This requires copying `QFormer.py`, `QFormer_Base.py`, `QFormer_output.py`, and `basemodel.py` into the MotionGPT codebase.
- **Integration pattern**: New architecture files as `lm` targets in config, new training stages in `MotionGPT` and `GPTLosses`.

### 2. `mGPT/archs/mgpt_masked_t2m.py` (NEW)
- **Adapted from**: `MaskTransformer` in `/home/student/hwu/Workplace/LaMP/models/mask_transformer/transformer.py`
- **Key class**: `MaskedT2MTransformer(nn.Module)`
- **Architecture**: Masked Transformer with iterative refinement + classifier-free guidance
- **Components**:
  - `token_emb`: `Embedding(514, 512)` — 512 VQ codes + MASK + PAD
  - `input_process`: Linear `[code_dim → latent_dim]`
  - `positional_encoding`: Sinusoidal
  - `cond_emb`: Linear `[768 → latent_dim]` (from QFormer output)
  - `transformer_encoder`: `nn.TransformerEncoder` with causal mask, 8 layers, 4 heads
  - `output_process`: BERT-style head → `[latent_dim → num_tokens]`
- **Text encoding**: Load frozen QFormer from LaMP checkpoint
  - Two-stage: BERT encode → project to 1408-dim → QFormer cross-attention → mean pool → `[B, 768]`
  - `text2former`: `Linear(768, 1408)` — **trainable** (only non-frozen part)
- **Training forward**:
  - BERT-style masking: 88% MASK, 10% random, 2% keep
  - Cosine masking schedule
  - CrossEntropy loss on masked positions
- **Generation**:
  - Iterative masked prediction (18 timesteps default)
  - Classifier-free guidance (scale=3)
  - Top-k filtering (threshold=0.9)
  - Cosine noise schedule
- **Compatibility methods**:
  - `__call__(texts, motion_tokens, lengths, tasks)` → Returns `LossOutput` compatible with `train_lm_forward()`
  - `generate_conditional(texts, lengths, stage, tasks)` → Returns `List[Tensor]` compatible with `val_t2m_forward()`
  - Uses Q0-only tokens: `motion_tokens[:, :, 0]` from RVQ codes
  - Decodes via `vae.decode_q0_only()`

## Architecture Comparison

| Aspect | Current (Hierarchical RVQ-GPT) | New (LaMP + Masked T2M) |
|--------|-------------------------------|------------------------|
| Text Encoder | CLIP/BERT (frozen) | QFormer (pretrained via LaMP, frozen for T2M) |
| Motion Tokenization | 3 quantizers (Q0,Q1,Q2) | 1 quantizer (Q0 only) |
| Generation Method | Autoregressive (3 decoders) | Iterative masked refinement (18 steps) |
| Text Conditioning | Direct projection | QFormer cross-attention → mean pool → condition vector |
| Guidance | None | Classifier-free (scale=3) |
| Pretraining | None (direct training) | LaMP contrastive pretraining |
| Training Objectives | 3 × CE (one per quantizer) | LaMP: 4 losses; T2M: masked CE |

## Risk Assessment

- **QFormer dependency**: Requires `bert-base-uncased` model/tokenizer. Must ensure HuggingFace access or local copy.
- **Distributed operations**: LaMP's `all_gather_with_grad` / `concat_all_gather` need care in single-GPU mode. Will add fallback.
- **Q0-only decoding**: The RVQ-VAE is trained with quantizer dropout, so `decode_q0_only()` should work, but quality may be lower than 3-quantizer decoding.
- **Memory**: LaMP's QFormer + BERT + motion encoder is ~200M params. Combined with the masked transformer, this is significant but manageable.
